## 1. Введение

### 1.1. Назначение системы

«ЮрДело» — система управления адвокатской практикой. Она позволяет вести учёт судебных дел, слушаний, документов, контрагентов и формировать отчёты.

### 1.2. Роли пользователей

В системе три роли:

| Роль | Описание |
|------|----------|
| **Руководитель** (admin) | Полный доступ ко всем разделам, управление пользователями, все отчёты |
| **Адвокат** (lawyer) | Работа со своими делами, персональные отчёты |
| **Наблюдатель** (viewer) | Просмотр всех дел без возможности редактирования |

### 1.3. Поддерживаемые браузеры

- Google Chrome 100+
- Mozilla Firefox 100+
- Microsoft Edge 100+
- Safari 15+

## 2. Вход в систему

### 2.1. Страница входа

При открытии системы отображается страница входа.

```
┌─────────────────────────────────────┐
│                                     │
│   ⚖  ЮрДело                        │
│                                     │
│   Email                             │
│   ┌───────────────────────────────┐ │
│   │ admin@yurdelo.ru              │ │
│   └───────────────────────────────┘ │
│                                     │
│   Пароль                            │
│   ┌───────────────────────────────┐ │
│   │ ••••••••                      │ │
│   └───────────────────────────────┘ │
│                                     │
│   ┌───────────────────────────────┐ │
│   │           Войти               │ │
│   └───────────────────────────────┘ │
│                                     │
└─────────────────────────────────────┘
```

**Действия:**
1. Введите Email, выданный администратором.
2. Введите пароль.
3. Нажмите кнопку «Войти».

### 2.2. Двухфакторная аутентификация (2FA)

Если для вашего аккаунта включена двухфакторная аутентификация, после ввода email и пароля появится дополнительное поле:

```
┌───────────────────────────────────┐
│ Код 2FA                          │
│ ┌───────────────────────────────┐ │
│ │ 123456                        │ │
│ └───────────────────────────────┘ │
└───────────────────────────────────┘
```

Введите 6-значный код из приложения-аутентификатора (Google Authenticator, Authy и т.д.).

### 2.3. Ошибки при входе

| Сообщение | Причина | Что делать |
|-----------|---------|------------|
| «Неверный email или пароль» | Неправильные учётные данные | Проверьте правильность ввода |
| «Аккаунт деактивирован» | Учётная запись отключена | Обратитесь к руководителю |
| «Аккаунт временно заблокирован» | 5 неудачных попыток входа | Подождите 15 минут |

## 4. Дела

### 4.1. Список дел

Раздел «Дела» содержит список всех доступных дел.

```
┌──────────────────────────────────────────────────────┐
│ Дела                                    [+ Дело]     │
│                                                      │
│ 🔍 Поиск дел...           [Все статусы ▾]            │
│                                                      │
│ Название          │ Истец      │ Ответчик  │ Статус   │
│ ──────────────────┼────────────┼───────────┼──────── │
│ Взыскание задолж. │ ООО Альфа  │ ИП Смирнов│ Активно  │
│ Банкротство ООО   │ ИП Смирнов │ ООО Бета  │ Закрыто  │
│ Трудовой спор     │ Козлов П.  │ ООО Гамма │ Активно  │
│                                                      │
│ Показано 3 из 15                    [1] [2] [3]      │
└──────────────────────────────────────────────────────┘
```

### 4.2. Поиск и фильтрация

- **Поиск:** введите текст в поле поиска. Поиск начинается автоматически после ввода 2 символов (с задержкой 300 мс).
- **Фильтр по статусу:** выберите «Активные», «Закрытые» или «Приостановленные».
- **Пагинация:** переключение между страницами внизу таблицы.

### 4.3. Создание нового дела

> Доступно для: **Руководитель**, **Адвокат**

**Способ 1:** Нажмите кнопку «+ Дело» на странице списка дел.

**Способ 2:** Перейдите по адресу `/cases/new`.

```
┌─────────────────────────────────────┐
│ Новое дело                     [✕]  │
│                                     │
│ Название дела                       │
│ ┌─────────────────────────────────┐ │
│ │ Взыскание задолженности         │ │
│ └─────────────────────────────────┘ │
│                                     │
│ Категория                           │
│ ┌─────────────────────────────────┐ │
│ │ Арбитраж                      ▾ │ │
│ └─────────────────────────────────┘ │
│                                     │
│ Истец                               │
│ ┌─────────────────────────────────┐ │
│ │ 🔍 Поиск...                     │ │
│ │ ООО Альфа (ИНН 1234567890)     │ │
│ │ ИП Смирнов                      │ │
│ └─────────────────────────────────┘ │
│                                     │
│ Ответчик                            │
│ ┌─────────────────────────────────┐ │
│ │ 🔍 Поиск...                     │ │
│ └─────────────────────────────────┘ │
│                                     │
│ Цена иска, ₽                       │
│ ┌─────────────────────────────────┐ │
│ │ 1500000                         │ │
│ └─────────────────────────────────┘ │
│                                     │
│          [Отмена]  [Создать]        │
└─────────────────────────────────────┘
```

**Поля формы:**

| Поле | Обязательное | Правила |
|------|-------------|---------|
| Название дела | Да | Минимум 3 символа |
| Категория | Да | Гражданское, Арбитраж, Административное, Уголовное, Трудовое |
| Истец | Да | Выбрать из списка контрагентов |
| Ответчик | Да | Выбрать из списка контрагентов |
| Цена иска | Нет | Число ≥ 0 |

> ⚠ Если истец и ответчик совпадают, отобразится предупреждение, но создание не блокируется.

После успешного создания вы будете перенаправлены на карточку нового дела.

```

│ ← Назад                                             │

│                                                      │
│ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ │
│ │ Статус   │ │Результат │ │Цена иска │ │ Создано  │ │
│ │ Активно  │ │Установить│ │ 1 500 000│ │15.03.2026│ │
│ └──────────┘ └──────────┘ └──────────┘ └──────────┘ │
│                                                      │
│ Стадии                                  [+ Стадия]   │

```

### 5.2. Редактирование дела

> Доступно для: **Руководитель** (любое дело), **Адвокат** (только своё дело)

Нажмите кнопку «Редактировать». Карточка переключится в режим редактирования с предзаполненной формой. После сохранения вы вернётесь в режим просмотра.

### 5.3. Смена статуса

> Доступно для: **Руководитель**, **Адвокат** (своё дело)

Нажмите на badge статуса с иконкой ▾. Появится выпадающее меню:

| Текущий статус | Доступные действия |
|---------------|-------------------|
| Активно | Закрыть дело, Приостановить |
| Закрыто | Возобновить |
| Приостановлено | Возобновить |

Каждое действие требует подтверждения через диалог.

### 5.4. Установка итогового результата

> Доступно для: **Руководитель**, **Адвокат** (своё закрытое дело)

В карточке «Результат» нажмите «Установить ▾» (или имя текущего результата для изменения):

| Результат | Описание |
|-----------|----------|
| Победа | Иск удовлетворён полностью |
| Проигрыш | В иске отказано |
| Частично | Иск удовлетворён частично |
| Мировое | Заключено мировое соглашение |

Каждый выбор требует подтверждения.

> ⚠ Результат **не устанавливается автоматически** при закрытии дела или создании слушания с результатом. Это всегда отдельное явное действие.

### 5.5. Удаление дела

> Доступно для: **Руководитель**

Нажмите иконку 🗑 и подтвердите удаление. Дело будет мягко удалено (soft delete) — данные сохраняются в БД.

### 5.6. Конфликт данных (STALE_DATA)

Если другой пользователь изменил дело, пока вы его редактировали, при сохранении появится диалог:

```
┌─────────────────────────────────────┐
│ Данные устарели                     │
│                                     │
│ Данные были изменены другим         │
│ пользователем. Обновите страницу,   │
│ чтобы получить актуальную версию.   │
│                                     │
│           [🔄 Обновить]             │
└─────────────────────────────────────┘
```

Нажмите «Обновить» — страница перезагрузится с актуальными данными. Ваши несохранённые изменения будут потеряны.

## 6. Стадии и слушания

### 6.1. Добавление стадии

> Доступно для: **Руководитель**, **Адвокат** (своё дело)

В карточке дела нажмите «+ Стадия».

```
┌─────────────────────────────────────┐
│ Новая стадия                   [✕]  │
│                                     │
│ Тип стадии                          │
│ ┌─────────────────────────────────┐ │
│ │ 1-я инстанция                 ▾ │ │
│ └─────────────────────────────────┘ │
│                                     │
│ Суд                                 │
│ ┌─────────────────────────────────┐ │
│ │ Арбитражный суд г. Москвы       │ │
│ └─────────────────────────────────┘ │
│                                     │
│ Номер дела                          │
│ ┌─────────────────────────────────┐ │
│ │ А40-12345/2025                  │ │
│ └─────────────────────────────────┘ │
│                                     │
│           [Добавить]                │
└─────────────────────────────────────┘
```

**Типы стадий:**

| Тип | Порядок |
|-----|---------|
| Досудебная | 1 |
| 1-я инстанция | 2 |
| Апелляция | 3 |
| Кассация | 4 |

> ⚠ Если вы добавляете стадию с порядком ниже уже существующей (например, «Досудебная» после «1-я инстанция»), появится жёлтое предупреждение. Это не блокирует создание — нажмите «Добавить» для продолжения.

### 6.2. Редактирование стадии

Нажмите иконку ✎ рядом с названием стадии. Можно изменить суд и номер дела.

### 6.3. Добавление слушания

> Доступно для: **Руководитель**, **Адвокат** (своё дело)

Внутри стадии нажмите «+ Слушание».

```
┌─────────────────────────────────────┐

│                                     │
│ Тип                                 │

│ └─────────────────────────────────┘ │
│                                     │

│ ┌─────────────────────────────────┐ │

│ └─────────────────────────────────┘ │
│                                     │
│ Заметки                             │
│ ┌─────────────────────────────────┐ │
│ │                                 │ │
│ └─────────────────────────────────┘ │
│                                     │
│           [Добавить]                │
└─────────────────────────────────────┘
```

### 6.4. Типы слушаний и их поля

| Тип | Описание | Дополнительные поля |
|-----|----------|---------------------|
| **Заседание** | Обычное судебное заседание | — |
| **Результат** | Вынесено решение | Результат (обязательно), Обжаловано |
| **Перенос** | Заседание перенесено | Новая дата (обязательно), Причина |
| **Заметка** | Произвольная запись | — |

### 6.5. Предложение обновить итоговый результат

При создании слушания типа «Результат» после сохранения появится синий баннер:

```
┌────────────────────────────────────────────────────┐
│ 💡 Обновить итоговый результат дела до «Победа»?   │
│                                    [Да]  [Нет]     │
└────────────────────────────────────────────────────┘
```

- **Да** — итоговый результат дела будет обновлён.
- **Нет** — баннер скроется, итоговый результат не изменится.

## 7. Документы

### 7.1. Просмотр документов

Документы отображаются под каждым слушанием:

```
│ Заседание · 15 мар 2026, 10:00
│   📄 contract.pdf (12 КБ)   [⬇] [🗑]
│   📄 decision.docx (45 КБ)  [⬇] [🗑]
│   [📤 Файл]
```

### 7.2. Загрузка файла

> Доступно для: **Руководитель**, **Адвокат** (своё дело)

1. Нажмите «📤 Файл» под слушанием.
2. Выберите файл в диалоге выбора.
3. Дождитесь окончания загрузки (отображается прогресс-бар).

**Ограничения:**
- Максимальный размер: **50 МБ**
- Допустимые форматы: PDF, DOCX, XLSX, JPEG, PNG, TIFF, TXT

При превышении 50 МБ файл не будет отправлен — ошибка появится до загрузки.

### 7.3. Скачивание файла

> Доступно для: **Все роли**

Нажмите иконку ⬇ рядом с файлом. Файл откроется в новой вкладке браузера через защищённую ссылку (действует 1 час). При каждом нажатии генерируется новая ссылка.

### 7.4. Удаление файла

> Доступно для: **Руководитель** (любой файл), **Адвокат** (только свои загрузки)

Нажмите иконку 🗑 и подтвердите удаление.

## 8. Передача дела

### 8.1. Просмотр истории передач

В нижней части карточки дела отображается секция «Передачи» с хронологией:

```
│ Передачи                        [Передать дело]
│ │ 15 мар 2026 · Петрова → Иванов
│ │ По согласованию
│ │
│ │ 10 фев 2026 · Иванов → Петрова
│ │ Передача на время отпуска
```

### 8.2. Передача дела другому адвокату

> Доступно для: **Руководитель**, **Адвокат** (только своё дело)

1. Нажмите «Передать дело».
2. Выберите получателя из списка активных адвокатов.
3. Укажите комментарий (необязательно).
4. Нажмите «Передать».
5. Появится предупреждение — нажмите «Подтвердить передачу».

```
┌─────────────────────────────────────┐
│ Передать дело                  [✕]  │
│                                     │
│ Кому                                │
│ ┌─────────────────────────────────┐ │
│ │ Иванов Алексей                ▾ │ │
│ └─────────────────────────────────┘ │
│                                     │
│ Комментарий                         │
│ ┌─────────────────────────────────┐ │
│ │ По согласованию                 │ │
│ └─────────────────────────────────┘ │
│                                     │
│ ⚠ Дело «Взыскание» будет передано  │
│   от Петровой к Иванову.            │
│   Продолжить?                       │
│                                     │
│       [Подтвердить передачу]        │
└─────────────────────────────────────┘
```

> ⚠ После передачи прежний адвокат теряет доступ к делу (если он не руководитель).

## 15. Безопасность

### 15.1. Пароли

- Минимальная длина: 8 символов.
- Обязательно: хотя бы одна буква и одна цифра.
- Пароли хранятся в зашифрованном виде (bcrypt).

### 15.2. Сессии

- Access token действует 15 минут, обновляется автоматически.
- Refresh token действует 7 дней, хранится в защищённой cookie.
- При смене пароля все сессии завершаются.

### 15.3. Блокировка при подборе пароля

После 5 неудачных попыток входа аккаунт блокируется на 15 минут.

### 15.4. Двухфакторная аутентификация

- Рекомендуется для всех пользователей.
- Обязательна для руководителей.
- Используется стандарт TOTP (RFC 6238).

## 17. Часто задаваемые вопросы

### Не могу войти в систему

1. Проверьте правильность email и пароля.
2. Если аккаунт заблокирован — подождите 15 минут.
3. Если аккаунт деактивирован — обратитесь к руководителю.

### Потерял доступ к приложению 2FA

Обратитесь к руководителю системы для сброса 2FA через базу данных.

### Не вижу раздел «Отчёты»

Раздел доступен только для руководителей и адвокатов. Наблюдатели не имеют доступа.

### Не могу редактировать чужое дело

Адвокаты могут редактировать только свои дела (где они назначены ответственными). Для доступа к чужому делу обратитесь к руководителю для передачи.

### При сохранении появляется «Данные устарели»

Другой пользователь изменил эти данные, пока вы работали с ними. Нажмите «Обновить» для загрузки актуальной версии и повторите свои изменения.

### Не могу загрузить файл

- Проверьте размер файла (максимум 50 МБ).
- Проверьте формат (PDF, DOCX, XLSX, JPEG, PNG, TIFF, TXT).
- При ошибке сети попробуйте ещё раз.

### Не могу удалить контрагента

Удаление невозможно, если контрагент используется в активных делах. Сначала закройте или удалите связанные дела.
