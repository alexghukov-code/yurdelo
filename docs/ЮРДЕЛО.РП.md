# 1. ОБЩИЕ СВЕДЕНИЯ О ПРОГРАММЕ

## 1.1. Назначение программы

Программный комплекс «ЮрДело» предназначен для автоматизации управления адвокатской практикой: учёта дел, слушаний, документов, контрагентов и отчётности.

## 1.2. Функции программы

- управление делами (создание, редактирование, смена статуса, передача);
- управление стадиями и слушаниями;
- хранение и скачивание документов (Selectel S3);
- управление пользователями (создание, деактивация, восстановление);
- ролевая модель доступа (admin, lawyer, viewer);
- отчётность (персональная и управленческая);
- двухфакторная аутентификация (TOTP).

## 1.3. Обозначение программы

**ЮРДЕЛО.01** — версия 2.1.0.

## 4.4. Установка Docker Engine

### 4.4.1. Добавление ключа репозитория

```bash
install -m 0755 -d /etc/apt/keyrings

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
  gpg --dearmor -o /etc/apt/keyrings/docker.gpg

chmod a+r /etc/apt/keyrings/docker.gpg
```

### 4.4.2. Добавление репозитория Docker

```bash
echo \
  "deb [arch=$(dpkg --print-architecture) \
  signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | \
  tee /etc/apt/sources.list.d/docker.list > /dev/null
```

### 4.4.3. Установка Docker и Docker Compose

```bash
apt update
apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
```

### 4.4.4. Проверка установки

```bash
docker --version
# Docker version 27.x.x

docker compose version
# Docker Compose version v2.x.x
```

### 4.4.5. Автозапуск Docker при загрузке сервера

```bash
systemctl enable docker
```

# 8. ПОЛУЧЕНИЕ SSL-СЕРТИФИКАТА

## 8.1. Первоначальное получение сертификата

Перед первым запуском необходимо получить сертификат Let's Encrypt.

### 8.1.1. Временный запуск nginx без SSL

Создать временный конфигурационный файл:

```bash
cat > /tmp/nginx-temp.conf << 'EOF'
server {
    listen 80;
    server_name yurdelo.ru api.yurdelo.ru;

    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    location / {
        return 200 'OK';
    }
}
EOF
```

### 8.1.2. Запуск временного nginx

```bash
docker run -d --name nginx-temp \
  -p 80:80 \
  -v /tmp/nginx-temp.conf:/etc/nginx/conf.d/default.conf \
  -v /opt/yurdelo/infra/nginx/ssl:/etc/letsencrypt \
  -v certbot_www:/var/www/certbot \
  nginx:alpine
```

### 8.1.3. Получение сертификата

```bash
docker run --rm \
  -v /opt/yurdelo/infra/nginx/ssl:/etc/letsencrypt \
  -v certbot_www:/var/www/certbot \
  certbot/certbot certonly \
    --webroot \
    --webroot-path=/var/www/certbot \
    -d yurdelo.ru \
    -d api.yurdelo.ru \
    --email admin@yurdelo.ru \
    --agree-tos \
    --no-eff-email
```

### 8.1.4. Создание символических ссылок

```bash
ln -sf /opt/yurdelo/infra/nginx/ssl/live/yurdelo.ru/fullchain.pem \
       /opt/yurdelo/infra/nginx/ssl/fullchain.pem

ln -sf /opt/yurdelo/infra/nginx/ssl/live/yurdelo.ru/privkey.pem \
       /opt/yurdelo/infra/nginx/ssl/privkey.pem
```

### 8.1.5. Остановка временного nginx

```bash
docker stop nginx-temp && docker rm nginx-temp
```

```bash
    --migration-file-language sql
```

```bash

  sh -c "

      -f db/seeds/001_stage_types.sql
  "
```

```bash
docker compose -f docker-compose.prod.yml up -d
```

## 11.1. Получение обновлений

```bash

cd /opt/yurdelo

```

## 11.2. Запуск процедуры обновления

```bash
cd infra
./scripts/deploy.sh
```

Скрипт выполняет следующие действия автоматически:

1. Сохраняет текущий тег для возможного отката.
2. Собирает новые Docker-образы.
3. Применяет миграции базы данных.
4. Запускает новую версию backend.
5. Проверяет healthcheck.
6. При неудаче — автоматический откат.

## 11.3. Проверка после обновления

```bash
curl -s https://yurdelo.ru/health | python3 -m json.tool
```

Убедиться, что поле `version` соответствует ожидаемому.

### 13.1.3. Настройка автоматического резервного копирования

# 15. СООБЩЕНИЯ СИСТЕМНОМУ ПРОГРАММИСТУ

## 15.1. Сообщения healthcheck endpoint

| Статус | HTTP | Значение | Действие |
|--------|------|----------|----------|
| `"status": "ok"` | 200 | Все компоненты работают | Норма |
| `"status": "degraded", "db": "error"` | 503 | PostgreSQL недоступен | Проверить контейнер postgres, логи, диск |
| `"status": "degraded", "redis": "error"` | 503 | Redis недоступен | Проверить контейнер redis, память |

## 15.2. Сообщения при развёртывании

| Сообщение | Причина | Действие |
|-----------|---------|----------|
| `Health failed! Auto-rollback...` | Новая версия не прошла healthcheck | Проверить логи: `docker compose logs backend` |
| `No previous tag — manual intervention required` | Первое развёртывание, откат невозможен | Исправить ошибку, повторить deploy |
| `Rollback OK` | Автооткат выполнен успешно | Исследовать причину сбоя новой версии |

## 15.3. Типичные ошибки и их устранение

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `ECONNREFUSED postgres:5432` | PostgreSQL не запущен | `docker compose up -d postgres`, дождаться healthy |
| `ECONNREFUSED redis:6379` | Redis не запущен | `docker compose up -d redis`, дождаться healthy |
| `certificate not found` | SSL-сертификат не получен | Повторить раздел 8 |
| `permission denied` | Неверные права на файлы | `chown -R yurdelo:yurdelo /opt/yurdelo` |
| `port 80 already in use` | Порт занят другим процессом | `lsof -i :80`, остановить конфликтующий процесс |
| `no space left on device` | Диск заполнен | `docker system prune -f`, удалить старые backup |
| `MIGRATION_DATABASE_URL not set` | Не настроен `.env` | Проверить раздел 7 |

## 15.4. Контактная информация

При возникновении проблем, не описанных в настоящем руководстве, обращаться к разработчику программного комплекса.
